**Yes. The log proves fee estimation is the immediate cause of the payment failure.**

At both payment attempts, ZNY-NOMP successfully constructed:

```text
sendmany "" {"TLRhJQw1VQfL94gwHxd8t5bBnEZuHoKZaW":296.9997}
```

but Qogecoin Core rejected it with:

```text
code: -6
Fee estimation failed. Fallbackfee is disabled.
```

## What this proves

This largely supersedes the earlier batch-order hypothesis for this instance.

The payment processor has already:

1. Read the pending blocks.
2. Called `gettransaction`.
3. Recognized mature blocks as `generate`.
4. Calculated the miner reward.
5. Reached the actual `sendmany` payment call.

The amount is also revealing:

```text
296.9997 QOGE
```

That corresponds to approximately:

```text
3 × 99 QOGE = 297 QOGE
297 − 0.0003 fee reserve = 296.9997 QOGE
```

So `maxBlocksPerPayment: 3`, the 99-QOGE pool output, and the transaction-fee reserve are all being applied correctly.

## Why the blocks remain pending

When `sendmany` returns an error, `paymentProcessor.js` calls:

```js
callback(true);
```

and stops before the final Redis update.

Therefore, these commands never run:

```text
SMOVE qogecoin:blocksPending qogecoin:blocksConfirmed ...
ZADD qogecoin:payments ...
```

The pending-to-confirmed move only happens after a successful payment cycle.

The subsequent:

```text
Finished interval
```

only means the payment interval ended. It does not mean payment succeeded.

## Fix it immediately

Set an explicit wallet transaction fee:

```bash
qogecoin-cli \
  -testnet \
  -rpcwallet=poolwallet \
  settxfee 0.0001
```

Expected:

```text
true
```

This should allow the next automatic 120-second payment attempt to proceed without waiting for fee estimation.

Then make it persistent in the actual `qogecoin.conf` loaded by systemd:

```ini
fallbackfee=0.0001
paytxfee=0.0001
```

For example:

```bash
nano /root/.qogecoin/qogecoin.conf
```

Then restart only the daemon:

```bash
sudo systemctl restart qogecoind
```

Wait until RPC is ready:

```bash
until qogecoin-cli -testnet getblockchaininfo >/dev/null 2>&1; do
    sleep 2
done
```

ZNY-NOMP should automatically retry at the next payment interval; it normally does not need restarting.

The successful line should resemble:

